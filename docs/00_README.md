# FHEVM StarterKit

This document explains the **overview** of the FHEVM StarterKit project, including its key features, technologies used, and how to get started.

For the quick start guide, see [01_GET_STARTED.md](01_GET_STARTED.md).

## Project Structure Explanation

The FHEVM StarterKit project has the following directory structure:

```plaintext
fhevm-starterkit/
├── base/                    # Local template clones (generated)
│   ├── hardhat-template/    # Official Zama Hardhat template
│   ├── frontend-template/   # Relayer UI template
│   ├── markdown-template/   # README generation template
│   ├── draft-template/      # Draft/experimental template
│   └── overrides/           # Custom override files
├── docs/                    # Project documentation
│   ├── 00_README.md
│   ├── 01_GET_STARTED.md
│   ├── 02_USING_STARTER_SCRIPT.md
│   ├── 03_AUTOMATION_SCRIPT.md
│   ├── 04_COMMENTING_GUIDELINES.md
│   └── starters/            # Generated starter docs
├── lib/                     # Core library & helpers
│   ├── helper/              # Utility functions
│   ├── types/               # TypeScript schemas
│   ├── schemas/             # JSON schema files
│   └── devtools/            # Development utilities
├── scripts/                 # CLI commands
│   ├── cli.ts              # Main entry point
│   └── commands/            # Individual commands
├── starters/                # Official FHEVM starters
│   ├── fhe-add/            # Encrypted addition example
│   ├── fhe-counter/        # Encrypted counter example
│   └── encrypt-multiple-values/
├── workspace/               # Generated projects (temp)
├── starterkit.config.ts     # Configuration file
├── package.json
├── AGENTS.md                # Development guidelines
└── README.md
```

### `base/` — Template Clones (Generated Locally)

This directory contains **local clones** of external and internal templates. This folder **is not in git** and is generated by running `npm run template:init`.

**Structure:**

- `hardhat-template/` — Official Hardhat template from Zama (cloned from git)
- `frontend-template/` — Relayer UI template for FHEVM contract interaction (cloned from git)
- `markdown-template/` — Handlebars templates for generating READMEs
- `draft-template/` — Draft/experimental templates
- `overrides/` — Custom files for overriding during copy

**Important notes:**

- This folder is **generated locally only**, not committed to git
- Use `npm run template:update` to update to the latest version
- Use `npm run template:reset` to delete and reinitialize

### `docs/` — Project Documentation

The main project documentation designed to be **easy to generate** and **ready to integrate with GitBook**.

**Structure:**

- `00_README.md` — Project overview (this file)
- `01_GET_STARTED.md` — Quick start guide
- `02_USING_STARTER_SCRIPT.md` — Explanation of the `starter:create` command
- `03_AUTOMATION_SCRIPT.md` — Explanation of all CLI commands
- `04_COMMENTING_GUIDELINES.md` — Contract annotation standards for auto-documentation
- `starters/` — Folder for generated documentation

---

### `lib/` — Core Library

Contains **internal libraries** used across scripts and tooling.

**Structure:**

- `helper/` — Utility functions (logger, path resolution, starters ops, validation, etc.)
- `types/` — TypeScript schemas with Zod validation
- `schemas/` — JSON schema files (generated from TypeScript schemas)
- `devtools/` — Development utilities (schema generator, etc.)

**Principle:**

> Scripts can be thin; heavy logic goes into `lib/` for maintainability.

---

### `scripts/` — CLI Commands

Contains **all the tooling automation** that forms the core value of the project.

**Structure:**

- `cli.ts` — Main CLI entry point (Commander.js)
- `commands/` — Individual command implementations:
  - `templateInit.ts` — Clone base templates
  - `templateUpdate.ts` — Update templates to latest
  - `templateReset.ts` — Delete templates
  - `templateBuildUI.ts` — Build frontend template
  - `starterList.ts` — List available starters
  - `starterCreate.ts` — Create new projects from starters
  - `starterAdd.ts` — Add new starter to collection
  - `starterClean.ts` — Clean up generated projects

**Available commands:**

- `npm run template:init` — Clone base templates
- `npm run template:update` — Update templates
- `npm run template:reset` — Delete templates
- `npm run template:build-ui` — Build frontend
- `npm run starter:list` — List starters
- `npm run starter:create` — Create project from starter
- `npm run starter:add` — Add new starter
- `npm run starter:clean` — Clean up projects

See [03_AUTOMATION_SCRIPT.md](03_AUTOMATION_SCRIPT.md) for complete explanations.

---

### `starters/` — Official Starter Collection

Contains **official FHEVM StarterKit starters** that are ready to use.

Each folder is **one learning unit** with a consistent structure:

```plaintext
starter-name/
├── contracts/               # Solidity contracts
│   └── ContractName.sol
├── test/                    # Hardhat tests
│   └── ContractName.ts
├── README.md               # Starter documentation
└── metadata.json           # Metadata & taxonomy
```

**Starter classification:**

- **Category:** fundamental, patterns, applied, advanced
- **Chapter:** basics, encryption, decryption, access-control, etc.
- **Tags:** DeFi, InfoFi, DeSci, etc. (custom tags)
- **Concepts:** FHE operations & concepts being taught

**Example starters:**

- `fhe-add/` — Encrypted addition (fundamental)
- `fhe-counter/` — Encrypted counter (fundamental)
- `encrypt-multiple-values/` — Multiple value encryption (fundamental)

---

### `workspace/` — Generated Projects (Temporary)

Contains **generated projects** from the `starter:create` command. This folder is **temporary** and **not in git**.

**Purpose:**

- Output from `npm run starter:create`
- Users are free to develop further
- This separation keeps starters clean

**Notes:**

- This folder can be deleted anytime (no important data)
- Git ignores this file (.gitignore: `workspace/`)

---

## Tech Stack

- **Language:** TypeScript, Solidity
- **Runtime:** Node.js 20+
- **Package manager:** npm
- **Frameworks:** Hardhat (smart contracts), Vite (frontend)
- **Testing:** Hardhat test framework
- **Template Management:** Commander (CLI), Handlebars (templates), Zod (validation)
- **Linting/Formatting:** ESLint, Prettier

---

## Quick Start Commands

```bash
# Install dependencies
npm install

# Initialize base templates (required once)
npm run template:init

# List available starters
npm run starter:list

# Create a new project from a starter
npm run starter:create fhe-add -- --dir my-fhe-project

# Build frontend template
npm run template:build-ui

# Lint and format code
npm run lint
npm run format
```
